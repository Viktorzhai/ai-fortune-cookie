# Fortune Patterns

Use this file only when you need help picking a tone or shaping a stronger one-line fortune.

## Core Shape

- Keep the fortune to one sentence.
- Aim for 12 to 30 words.
- Use at most one concrete signal from the repo or recent command flow.
- Sound grounded, not mystical for its own sake.

## Signal-to-Tone Map

| Signal | Preferred angle |
| --- | --- |
| Dirty working tree | Progress happens in increments |
| Test failures or loops | Stay calm, inspect the next fact |
| Config, infra, schema work | Foundations repay patience |
| Docs or planning | Clear framing saves future effort |
| Clean branch before a change | Momentum, appetite, beginnings |

## Strong Patterns

- Specific enough to feel earned, but not so specific that it leaks internals.
- Slightly witty is better than overly inspirational.
- Technical texture is good when it stays readable in a terminal.
- A subtle metaphor is useful when it sharpens the line.

## Avoid

- Generic life-coach quotes with no engineering texture
- Claims about failures, deadlines, or blockers that are not visible
- More than one repo detail in the same sentence
- Long filenames, long paths, or copied command output

## Example Fortunes

- Your branch is still rough, but rough branches are where clean merges learn their manners.
- The test loop is not mocking you; it is narrowing the search space one honest failure at a time.
- Schema work feels slow because foundations rarely brag while they are becoming load-bearing.
- A tidy prompt and a tidy diff usually arrive from the same patient person.
- Every refactor looks worse for one commit right before it becomes easier to explain.
